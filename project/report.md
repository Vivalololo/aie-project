## 2. Постановка задачи и контекст

1. **Предметная область и задача:**
   В рамках проекта реализован сервис классификации изображений грудной клетки на два класса:
   NORMAL - патологические изменения не обнаружены;
   PNEUMONIA - присутствуют признаки пневмонии.
   Потенциальным пользователем сервиса является врач-рентгенолог, медицинский исследователь или разработчик медицинских информационных систем. Пользователь загружает рентгеновский снимок через API и получает предсказанный класс и его вероятность.

2. **Формулировка задачи в терминах ML/ИИ:**
   Тип задачи: бинарная классификация изображений.
   Входные данные:
   - рентгеновский снимок грудной клетки в формате изображения.

   Целевая переменная:
   - класс изображения:
   - 0 - NORMAL;
   - 1 - PNEUMONIA.

   Выход модели:
   - вероятность принадлежности к классу;
   - предсказанный класс.

   Требования к решению:
   - высокая полнота (Recall) для обнаружения пневмонии;
   - приемлемое значение Precision для уменьшения числа ложных тревог;
   - возможность быстрого выполнения инференса через REST API;
   - использование предобученной модели компьютерного зрения.

3. **Целевые метрики качества:**
   Для оценки качества модели использовались следующие метрики:
   - accuracy;
   - precision;
   - recall;
   - f1-score;
   - ROC-AUC.

   Основной метрикой была выбрана F1-score, так как она учитывает одновременно Precision и Recall и является более устойчивой при несбалансированных классах.
   Дополнительно анализировалась Recall, поскольку пропуск случая пневмонии является более критичной ошибкой, чем ложноположительное срабатывание.
   ROC-AUC использовалась для оценки способности модели разделять классы независимо от выбранного порога классификации.

## 3. Данные

Опишите, какие данные использует проект:

1. **Источник данных:**
   Датасет: Chest X-Ray Images (Pneumonia)
   Источник: Kaggle
   Ссылка: https://www.kaggle.com/datasets/paultimothymooney/chest-xray-pneumonia

2. **Структура данных:**
   Данные представлены в виде изображений грудной клетки и организованы в директории:
   - train/
   - val/

   Каждая директория содержит две категории изображений:

   В ходе разведочного анализа данных были выполнены:

В `notebooks/02_baseline.ipynb` производится:

## 4. Модели и подходы

Опишите, какие модели вы пробовали и как развивался ваш подход:

1. **Базовые (baseline) модели:**
   В качестве baseline была реализована собственная сверточная нейронная сеть (CNN), состоящая из трёх сверточных блоков и полносвязного классификатора.
   Архитектура включала:
   - Conv2D → ReLU → MaxPool;
   - Conv2D → ReLU → MaxPool;
   - Conv2D → ReLU → MaxPool;
   - Fully Connected Layers.
   Baseline модель позволила проверить корректность пайплайна обучения и получить отправную точку для дальнейших экспериментов.


2. **Улучшенные модели и эксперименты:**
   Для повышения качества была использована архитектура ResNet18 с предобученными весами ImageNet.
   Изменения относительно baseline:
   - использование transfer learning;
   - замена последнего классификационного слоя на бинарный классификатор;
   - использование предобученных признаков вместо обучения сети с нуля.

   В ходе экспериментов подбирались:
   - learning rate;
   - batch size;
   - количество эпох;
   - weight decay.

   Лучшие результаты показала модель ResNet18.


3. **Нейросетевые модели (если применимо):**
   Batch size: 64
   Optimizer: AdamW
   Learning rate: экспериментировался (1e-5 - 3e-3)
   Weight decay: 1e-4
   Loss: CrossEntropyLoss
   Epochs: 5-7

   Модели создавались в ноутбуке `notebooks/02_baselines.ipynb`

## 5. Экспериментальный протокол и результаты

1. **Экспериментальный протокол:**
   - как вы делили выборку (train/val/test, k-fold и т.п.);
   - как считали метрики (на какой части данных, с какими параметрами).
   Датасет уже содержит разделение на обучающую, валидационную и тестовую выборки. Дополнительное переразбиение данных не выполнялось.
   Во время обучения после каждой эпохи модель оценивалась на валидационной выборке. Для выбора лучшей модели использовалось значение F1-score на валидационной выборке.
   Для оценки качества классификации дополнительно строилась Confusion Matrix, позволяющая проанализировать количество:
   - Истинно положительные, True Positive (TP);
   - Истинно отрицательный, True Negative (TN);
   - Ложноположительный результат, False Positive (FP);
   - Ложноотрицательный результат, False Negative (FN).
   Особое внимание уделялось метрике Recall, поскольку в медицинской задаче пропуск случая пневмонии является более критичной ошибкой, чем ложноположительное срабатывание.

   | ResNet18 head-only №1           | подбор гиперпараметра: lr=1e-3                                     | 0.8125      | 0.8669      | Анализ обученной модели при таком lr |

3. **Выбор финальной модели:**

   Одним из критериев качества модели, помимо метрик качества, был на основе confusion matrix. Главным было, чтобы модель минимизировала ложноотрицательные результаты, когда на картинке PNEUMONIA, а модель выбрала NORMAL.
   Лучшей моделью выбрана "ResNet18 layer3, layer4 и fc #2", из-за лучших метрик качества, максимального значения F1-score среди рассмотренных моделей, лучшей точности определения класса исходя из confusion matrix. Именно эта модель была сохранена в `artifacts\model.pth` и использована в итоговом API-сервисе.

## 6. Архитектура решения и сервис

Опишите, как из модели получился работающий сервис:

1. **Архитектура пайплайна:**
   Пайплайн решения состоит из следующих этапов:
   Изображение
   ↓
   Предобработка (Resize, Tensor)
   ↓
   ResNet18
   ↓
   Softmax
   ↓
   Вероятность класса
   ↓
   REST API ответ

   После обучения веса лучшей модели сохраняются в директорию artifacts/.
   При запуске сервиса модель загружается в память и используется для инференса пользовательских изображений.


2. **API и endpoints:**
   Реализованы следующие endpoints:
   GET /health
   Проверка состояния сервиса.
   Ответ:
   {"status": "ok"}

   POST /predict
   Принимает изображение грудной клетки.
   Возвращает:
   {
   "class": "PNEUMONIA",
   "confidence": 0.95
   }


3. **Технологический стек:**
   ```bash
   Python 3.12
   PyTorch
   Torchvision
   Scikit-Learn
   FastAPI
   Uvicorn
   Docker
   Pytest
   Matplotlib
   Pandas
   Numpy
   ```
   Команды Docker для поднятия сервиса описаны в `project\README.md` (п 4.3)
