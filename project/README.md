## 1. Паспорт проекта

- **Название проекта:** `Сервис для обнаружения патологии на рентгеновских снимках грудной клетки.`
- **Автор:** `Нарских Владимир Андреевич`
- **Группа:** `ИНБО-31-24`
- **Контакт:** `tg:@bonakvaviva / narskikh.v.a@edu.mirea.ru`

- **Краткое описание:**
  > Проект посвящен построению сервиса для обнаружения патологии на рентгеновских снимках грудной клетки. 
  > Используются открытый датасет Chest X-Ray Images (Pneumonia) (Kaggle), baseline-модели
  > В результате сервис работает на Fast API в контейнере Docker. Пользователь загружает изображение на сервер и получает результат работы модели: найдена патология или нет.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта для запуска сервиса (библиотеки Python, необходимые для компилирования Docker образа).
- `requirements-dev.txt` – зависимости проекта для работы с моделями (библиотеки Python, необходимые для запуска ноутбуков, train).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса.
- `data/` – демонстрационные/учебные данные:
  - есть `test_image_jpeg` для тестирования /predict эндпоинта
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённая лучшая модель, графики обучения разных моделей, `confusion_matrix.png` матрица ошибок лучшей модели.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.12.5`,
- Docker
- CUDA (NVIDIA Container Toolkit)

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate

# Linux / macOS:
source .venv/bin/activate

# Установить зависимостей
pip install --upgrade pip
pip install -r requirements-dev.txt
```

---

## 4. Как запустить проект

### 4.1. Запуск jupyter-сервера для работы с ноутбуками
```bash
cd project
source .venv/bin/activate
jupyter notebook
```

### 4.2. Запуск обучения модели

```bash
cd project
source .venv/bin/activate
python -m src.train      
```

### 4.3. Запуск сервиса через Docker (API/веб-интерфейс)

```bash
cd project
docker build -t chest-xray-api .
docker run -p 8000:8000 chest-xray-api
```
Сервер поднимается на 8000 порту и открывается в браузере по локальному адресу localhost:8000

Эндпоинты:
- docs/ (GET) - документация сервиса
- health/ (GET) - проверка состояния сервиса
- predict/ (POST) - принимает изображение грудной клетки

## 8. Ограничения и дальнейшая работа

В текущей версии сервиса реализована только классификация изображения.
В дальнейшем можно создать модель не только для классификации изображения на классы, но и реализовать детектирование области найденной патологии.
